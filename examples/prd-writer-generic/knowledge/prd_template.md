# PRD 模板参考

## 标准结构

一份合格的 PRD 应包含以下部分：

### 1. 文档信息
- 文档版本
- 作者 / 审核人
- 创建日期 / 最后更新

### 2. 背景与目标
- 业务背景
- 核心目标（不超过 3 条）
- 成功指标

### 3. 用户故事
格式：作为 [角色]，我希望 [功能]，以便于 [价值]

### 4. 功能需求
- 功能列表（表格形式：编号、名称、描述、优先级）
- 业务流程图（文字描述）
- 异常流程处理

### 5. 非功能需求
- 性能要求
- 安全要求
- 兼容性要求

### 6. 验收标准
每个功能点至少一条可测试的标准。

### 7. 里程碑
- 阶段划分
- 关键时间节点

### 8. 风险与依赖
- 技术风险
- 外部依赖
- 缓解方案
